# 🇺🇦 BG3 Voice Reader: Real-time OCR & TTS
Цей проект - це асистент для відеоігор (зокрема Baldur's Gate 3), який у реальному часі захоплює обрану ділянку екрану, розпізнає український текст діалогів за допомогою нейромережі та озвучує його природним голосом повністю офлайн.
Програма побудована на багатопроцесорній архітектурі (Multiprocessing), що дозволяє виконувати "важкі" матричні обчислення комп'ютерного зору та синтезу мовлення паралельно, не блокуючи роботу гри та усуваючи затримки звуку.
Демо: https://youtu.be/b9TWa4wSOng 

### 🚀 Основні можливості
- Високошвидкісне захоплення екрану: Використання бібліотеки dxcam з бекендом winrt для зчитування кадрів напряму з буфера відеокарти без впливу на FPS гри.
- Оптичне розпізнавання (OCR): Інтеграція EasyOCR з попередньою жорсткою фільтрацією зображень через OpenCV (збільшення масштабу, розмиття за Гауссом, бінаризація Оцу) для точного розпізнавання ігрових шрифтів.
- Фільтрація скролінгу: Використання алгоритму Раткліффа/Обершельпа (difflib.SequenceMatcher) для порівняння подібності рядків та відсікання "старого" тексту під час прокрутки списку відповідей.
- Офлайн TTS: Природний український голос, що генерується локально за допомогою нейромережі ukrainian-tts із підтримкою автоматичної розстановки наголосів.
- Глобальні гарячі клавіші: Апаратне перехоплення натискань на рівні ОС для миттєвого керування озвучкою навіть у повноекранних іграх.

### 🛠 Технологічний стек
- Python 3.12 (рекомендовано для сумісності прекомпільованих модулів C++)
- PyTorch (CUDA) - апаратне прискорення нейромереж на відеокартах NVIDIA.
- OpenCV-Python — комп'ютерний зір та інтерактивне виділення зон cv2.selectROI.
- DXcam — захоплення екрану (Windows Graphics Capture).
- Pygame-CE — асинхронне відтворення аудіо без блокування потоку.

### 🎮 Як користуватися
- Запустіть гру та переконайтеся, що в налаштуваннях відео встановлено режим "Вікно без рамок" (Borderless Window). Це необхідно для коректного захоплення екрану.
- Запустіть скрипт:
- На екрані з'явиться поточний кадр гри. За допомогою миші виділіть прямокутну зону, де з'являтиметься текст (діалоги або субтитри). Намагайтеся виділити якомога меншу ділянку для економії відеопам'яті та пришвидшення OCR.
- Натисніть SPACE (Пробіл) або ENTER, щоб зберегти координати зони.
- Поверніться до гри. Програма автоматично моніторитиме зону і читатиме новий текст.

### ⌨️ Гарячі клавіші
Ctrl + Numpad 0: Миттєва зупинка (Mute) — зупиняє озвучення поточного речення та очищає чергу (корисно, якщо ви вже прочитали текст).
Ctrl + Numpad 9: Пауза моніторингу — призупиняє роботу модуля комп'ютерного зору.
Ctrl + Numpad 8: Вихід — безпечно завершує всі фонові процеси.

### ⚠️ Відомі проблеми та їх вирішення
- Чорний екран замість гри (DXcam повертає порожнечу):
Виникає при роботі в ексклюзивному повноекранному режимі. Переведіть гру в "Borderless Window". Якщо проблема залишається, вимкніть функцію "Планування графічного процесора з апаратним прискоренням" (HAGS) в налаштуваннях дисплея Windows 11.
- Велика затримка (Latency) при появі тексту:
Моделі EasyOCR та ukrainian-tts споживають значний обсяг відеопам'яті (VRAM). Якщо відеокарті не вистачає ресурсів для паралельної роботи гри (наприклад, на налаштуваннях "Ультра"), швидкість падає. Знизьте якість текстур у грі або встановіть параметр gpu=False для EasyOCR у коді, щоб перенести розпізнавання тексту на центральний процесор.
- Голос повторює одну фразу нескінченно:
Виникає через засмічення черги дрібними змінами фону гри. У проекті реалізовано алгоритм SequenceMatcher, який фільтрує шум розпізнавання, ігноруючи текст, якщо його подібність до попереднього кадру становить понад 85%.

- режжим читання за наведенням курсора текст в зоні курсора
- оптимізація та квантування. пришвидшення роботи
- Форки на

  - вузьке направлення, для оакремих ігор прибрати читання через  розпізнавання тексту і вивантажувати його з напряму з оперативної памяті в модуль озвучення
- розширити набір гарячих клавіш  та управління (читати нове, обрану відповідь, всі відповіді і т.п.)
